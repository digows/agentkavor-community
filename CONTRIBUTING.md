# Contributing to Kavor

Kavor is early. Concrete workflows, sharp questions, and honest failure reports are more useful than formal
proposals.

## Choose the right place

- Use **Building Kavor** Discussions to react to an active experiment or challenge one of its assumptions.
- Use **Ideas** for a problem or workflow Kavor does not address yet.
- Use **Q&A** when you need help understanding or using Kavor.
- Use **Show and tell** to share a workflow, result, or lesson.
- Use **Issues** only for reproducible product bugs.
- Use **Private Vulnerability Reporting** for security concerns.

Search before posting. If a conversation already covers the subject, vote and add the missing context instead of
starting a duplicate.

## Useful contributions

The most helpful contributions usually include at least one of these:

- the real workflow you are trying to improve;
- what you expected to happen;
- the current workaround;
- evidence such as a sanitized screenshot or minimal reproduction;
- a trade-off the current discussion may have missed;
- feedback from trying a Kavor build or public experiment.

You do not need a complete solution. A well-described problem is enough.

## Language

English is the default language for maintainer-authored content. Contributions and conversations in other languages
are welcome.

## Public information

Everything posted in Discussions, Issues, and pull requests is public. Remove credentials, private source code,
unsanitized terminal output, personal data, local paths, repository details, and confidential Workspace information
before submitting.

## Pull requests

This repository contains Kavor's public community content and configuration, not the product source. Pull requests
may improve documentation, Building Kavor material, participation forms, and community configuration.

Keep a pull request focused and explain:

1. what problem it solves;
2. what changed;
3. how you checked links, formatting, or other affected behavior.

Product implementation changes are not accepted in this repository.

By participating, you agree to follow the [Code of Conduct](CODE_OF_CONDUCT.md).
