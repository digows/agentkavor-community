# Building Kavor

Building Kavor is a public engineering notebook about the product decisions behind Kavor.

It is not roadmap theater and it is not a stream of polished announcements. Each track starts with a real problem,
shows experiments and dead ends, and records what the evidence changed.

## Current track

### 01 — Visible Agent Capabilities

Coding agents already collect instructions, tools, plugins, MCP servers, permissions, and provider-specific
configuration. Most of that power is invisible at the moment it matters.

We are exploring how Kavor can make an agent's capabilities visible, attachable, and revocable directly from its
Node—without collapsing instructions, executable tools, credentials, and authorization into one vague concept.

[Follow the Visible Agent Capabilities track](visible-agent-capabilities/README.md).

## How a track works

Each track is developed in small chapters:

1. State the problem without pretending the solution is known.
2. Show the current product and technical constraints.
3. Build the smallest experiment that can invalidate an idea.
4. Share the result and ask a concrete question.
5. Record the decision, including what was rejected and why.

The durable summary lives in this repository. Discussions are where the community challenges assumptions, shares
workflows, and helps choose the next experiment.
